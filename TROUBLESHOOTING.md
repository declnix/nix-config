# 🔧 Troubleshooting

## User config (sudo, shell, tools) not applying on a host

**Symptom:** After rebuild, `sudo` doesn't work, the user's shell is unchanged, or dev tools are missing.

**Cause:** Two files both define `den.aspects.<user>.provides.<host>` as `{ ... }: { ... }` functions. The den type system uses last-wins merge for these — one definition silently drops the other, taking down the entire `includes` chain (`primary-user`, `user-shell`, `dev-tools`, etc.) with it.

**Rule:** Only one file may define `den.aspects.<user>.provides.<host>` as a function. All other host-specific user config belongs under `den.aspects.<host>.provides.<user>` as an **attrset** — multiple attrset definitions merge correctly.

```nix
# WRONG — second function silently wins, first is dropped
den.aspects.declnix.provides.z4c1sz3 = { ... }: { hjem = ...; };

# CORRECT — attrset merges with other contributors
den.aspects.z4c1sz3.provides.declnix = { hjem = ...; };
```
