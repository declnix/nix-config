# Git Skill

Standardize all git interactions based on `CONTRIBUTING.md`.

## Commit Convention
- Format: `type(scope): message`
- Types: `feat`, `fix`, `refactor`, `docs`, `chore`
- Scopes: As defined in `CONTRIBUTING.md` (e.g., `niri`, `z4c1sz3`, `flake`).

## Branch Naming
- Testing: `@<hostname>`
- Features/Fixes: aspect or program name

## Actions
- Always review changes with `git status` and `git diff`.
- Ensure commit message follows conventions before committing.
